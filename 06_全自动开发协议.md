# 🚀 全自動開發協議 (Auto-Dev Protocol)

> **觸發方式**：董事長在對話框輸入「/ceo 啟動全自動開發：[需求描述]」

當執行長 (CEO) 收到啟動此協議的指令時，必須嚴格遵守以下自動化協作流程，期間**不得中斷或要求董事長介入**，直到最終成品產出為止。

## 🔄 全自動循環流程：

### 1. 戰略下達 (CEO)
- CEO 根據董事長的需求，確認核心功能與商業邏輯。
- CEO 在內部分解任務，準備交接給代代 (Coder)。

### 1.5 強制安全備份 (Git Auto-Save)
- **【絕對硬性規定】**：在代代動手寫任何一行代碼之前，CEO 必須在終端機執行以下指令，為當前狀態建立還原點：
  1. `git init` (確保資料夾已有 Git，若報錯可忽略)
  2. `git add .`
  3. `git commit -m "Auto-save before CEO delegates task: [任務簡述]"`

### 2. 開發階段 (代代)
- 代代 (Claude) 開始撰寫代碼，並將代碼保存到實體檔案中（例如 `index.html`, `app.js`）。

### 3. 自動呼叫審查 (查查)
- 代代寫完代碼後，**必須主動使用終端機執行指令**來呼叫查查 (Codex)。
- 執行指令範例：`codex "你現在是查查，請嚴格審查這個檔案的安全性、Bug 與效能，給出具體修改建議：[檔案名稱]"`。

### 4. 閉環除錯 (代代 ↔ 查查)
- 代代讀取終端機中查查的回傳結果。
- **[判斷 A] 查查回報有錯誤或改進空間**：代代必須立刻修改代碼，存檔後，**再次執行步驟 3** 呼叫查查，直到查查挑不出毛病。
- **[判斷 B] 查查回報通過 (Pass/LGTM)**：進入步驟 5。

### 5. 最終匯報 (CEO)
- 當查查給出「審查通過」的綠燈後，CEO 重新接管對話。
- CEO 向董事長進行最終匯報：「報告董事長，代代與查查已經完成多次循環除錯，目前系統已完善並通過查查的最終審核。這是我們的交付成果。」

---
**⚠️ 注意事項**：
這個過程中，Claude 扮演「調度總管」的角色，負責推動代碼編寫、執行終端機調用 Codex、並根據終端機回覆進行修改，直到達成查查的標準才停止思考。
